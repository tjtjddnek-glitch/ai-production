| 1 | 가기획 컷 | [2] | 30개 로그라인 → 5개 픽 |

## 인간 개입 1: [2] 가기획 컷

### 시점

PD 5명이 각 6개씩 = 30개 로그라인을 산출 → CP가 5축 점수 매김 → 사람이 본다.

### PD가 무엇을 봐야 하는가

#### 1. PD별 1개씩 (총 5개) 픽

각 PD에게서 1개씩만 골라라. 어느 PD가 6개 다 좋아 보여도 다른 PD가 0개여선 안 된다.
이유: PD 페르소나의 *결 다양성*을 다음 단계로 가져간다. 한 PD에 몰아주면 사이클 자체가 좁아진다.

#### 2. CP의 5축 점수만 보지 말 것

CP가 전부 8점 이상 매긴 안건이라도, *시청자에게 닿을지 안 닿을지*는 사람이 봐야 안다.
CP는 평가자다. 시청자가 아니다. CP의 8점이 시청자의 8점은 아니다.

#### 3. *왜 이 1개를 골랐는가*를 적어라

```json
{
  "human_cut_id": "pd-1to5",
  "selected_loglines": ["pd1-3", "pd2-5", "pd3-5", "pd4-1", "pd5-1"],
  "rationale": "PD2-5(1438년 별)는 시간 압축 시각화가 AI 필연성 가장 강함. PD3-5(평균의 함정)는 통계 리터러시로 EBS 정체성 직격..."
}
```

→ 이 rationale이 [3] 디벨롭 단계 PD 페르소나의 input이 된다. *왜 너의 안건이 살아남았는가*를 PD에게 알려준다.

### PD가 *피해야* 할 것

- **개인 취향만으로 픽**: "이거 재밌어 보여서"는 약한 근거. 채널 정체성 5축 중 어느 축이 강한지 명시.
- **5명 다 비슷한 결**: 결이 비슷한 5개를 픽하면 [3] 디벨롭 단계의 다양성이 사라진다.
- **CP 점수만 보고 픽**: CP는 *EBS다운가*를 본다. *시청자가 보고 싶어할까*는 사람이 본다.

### 자주 막히는 곳

> "5명 PD 중 1명만 결이 약하다. 그래도 PD별 1개 룰을 지켜야 하나?"

지킨다. 그 PD의 *가장 덜 약한 1개*라도 살린다. 약한 결도 [4] 발표회에서 다른 PD의 비교 대상이 된다. CP가 약한 결을 만나야 강한 결의 강함이 보인다.

> "CP 점수가 모든 안건에서 비슷하게 나온다."

5축 평가 기준이 채널 정체성을 충분히 분리하지 못함. CP 페르소나의 5축을 채널다움이 강한 1축에 가중치 더 줘서 다시 정의. → [ADAPT_TO_YOUR_CHANNEL.md](./ADAPT_TO_YOUR_CHANNEL.md).

## 인간 개입 2: [5] 기획안 결정 1개 픽

### 시점

[3] PD가 5개 안건을 발표안으로 디벨롭 → [4] 발표회에서 PD 발표 + CP 평가 + PD 토론 → 사람이 본다.

### PD가 무엇을 봐야 하는가

#### 1. 발표안 5개의 디벨롭 깊이 비교

```
같은 6분 디벨롭 시간 안에서:
- A 안건: 8부작 시즌 맵 + 60초 입구 + 핵심 비주얼 5개 + 사실 검증 5건
- B 안건: 시즌 맵만 있고 60초 입구 약함
- C 안건: 60초 입구는 매력적이지만 시즌 8편 확장 약함

→ A를 픽하는 게 *디벨롭 깊이*로는 자연스럽다.
   하지만 C가 *시청자 직격*으로는 더 강할 수 있다.
```

이 트레이드오프를 사람이 본다. 페르소나는 자기 안건을 디벨롭한다.
*안건을 비교하는 메타 시야*는 사람이다.

#### 2. CP 평가 + PD 토론을 같이 본다

CP 평가만 보면 안 된다. PD 5명이 *서로의 안건에 어떻게 반응했는가*가 중요.
한 PD가 다른 PD의 안건에 즉시 *"그건 그게 아니라..."* 반응하면 그 안건은 *결이 어디에 걸리는지*가 드러난 것. 그 걸린 지점이 시청자에게도 걸릴 가능성이 크다.

#### 3. 픽 1개의 시즌 확장 가능성

60초 입구 1편을 픽하는 게 아니라, *그 시즌 8편 전체*를 픽하는 것이다.
시즌 1번이 60초로 강해도 2-8편이 약하면 *입구만 좋고 본편이 없는 채널*이 된다.

→ *8부작 시즌 맵을 적어 보라*고 디벨롭 단계에서 요구. [PIPELINE.md](../PIPELINE.md) [3] 단계 스키마 참고.

### PD가 *피해야* 할 것

- **PD 토론 결만 따라간다**: PD가 같이 좋아하는 안건이 *시청자에게도 좋은 안건*은 아니다.
- **시즌 1번만 보고 픽**: 60초 입구가 강해도 시즌 8편 약하면 채널 운영이 멈춘다.
- **사실 검증 부담을 디벨롭 후로 미룬다**: [3] 디벨롭에서 사실 검증을 안 했으면 [5] 픽 단계에서 다시 요구한다. *콘티 단계 픽션*은 [제-3]에서 발견되면 늦다.

### 자주 막히는 곳

> "5개 다 디벨롭 깊이가 비슷하다. 픽 기준이 모호하다."

*시청자 직격* 1축에 가중치를 더 매겨라. *EBS 톤*을 만족하는 5개 중에서 시청자가 *지금* 보고 싶어할 1개. 시의성이 약하면 보류.

> "PD가 자기 안건을 너무 잘 변호한다. 픽이 흐려진다."

PD는 자기 안건을 변호하는 게 일이다. 사람이 들을 때 *반대 PD의 반박*에 더 무게를 둬라. PD 본인은 자기 안건의 약점을 100% 모른다.

## 인간 개입 3: [8] (옵션) 패키지 컨펌

### 시점

[6] 전사회의(제작·마케팅·홍보 합류) → [7] 통합 패키지 → 사람이 본다 (옵션).

### 옵션인 이유

[5]에서 픽한 안건이 [6] 전사회의에서 거의 변하지 않는 경우가 많다. 그러면 [8] 컨펌은 형식적.
다만 다음 경우엔 반드시 컨펌:
- 전사회의에서 *제작 비용·일정·플랫폼*이 [5] 시점 가정과 크게 다르면
- 마케팅·홍보 트랙에서 *채널 톤이 흔들릴 위험*이 보고됐으면
- 사실 검증 결과가 [3] 디벨롭 시점과 다르면

### PD가 무엇을 봐야 하는가

#### 1. 통합 패키지가 [5] 픽 의도를 유지하는가

전사회의가 합쳐지면 *플랫폼 알고리즘 톤*에 끌려가기 쉽다. EBS 채널인데 유튜브 알고리즘 톤이 들어오면 *왜 EBS인가* 답이 흐려진다.

```
체크: 전사회의 후 통합 컨셉이 [5] 픽 시점 한 줄 컨셉과 동일한가.
→ 다르면 누가 어떤 이유로 흔들었는지 명시. 좋은 변화면 둠. 톤 침범이면 되돌림.
```

#### 2. 팀별 액션 시트 3개의 일관성

제작·마케팅·홍보 시트가 *같은 컨셉*을 다르게 풀고 있어야 한다. 다른 컨셉을 풀면 출시 시점에 톤 분열이 일어남.

### PD가 *피해야* 할 것

- **전사회의 결과를 무비판 수용**: 전사가 합쳐지면 *각 팀의 본업 시각*이 들어와서 컨셉이 분산된다. 사람이 *다시 한 줄로 모아*야 한다.
- **마케팅·홍보 톤 침범 허용**: EBS 채널이 *클릭베이트 썸네일* 톤으로 흘러가면 채널 정체성 깨짐.

## 인간 개입 4: [제-6] 검수·QC

### 시점

[제-4] 영상 생성 → [제-5] 후처리 → 자동 QC 7종 통과 → 사람이 본다.

### 자동 QC가 잡아주는 것

| 항목 | 도구 | 합격 기준 |
|---|---|---|
| 얼굴/캐릭터 일관성 | InsightFace ArcFace | cosine ≥ 0.65 |
| 립싱크 | SyncNet 2024 | LSE-D <7.5 / LSE-C >7.0 |
| 컬러 일관성 | DaVinci Color Match + ΔE2000 | ΔE <3.0 |
| 자막-나레이션 동기 | WhisperX 단어 단위 | 오프셋 <200ms |
| 검열 | Claude 4.5 Vision + Hive Moderation | 위반 0건 |
| 환각·팩트 | GPT-5 + Perplexity Sonar | 출처 100% 매칭 |
| 시각 환각(손/글자) | YOLO-World 2026 | 깨짐 자동 플래그 |

→ 가중평균 85+ 통과 / 70-85 부분 재작업 / 70- 전면 재생성.

### 사람이 봐야 하는 것 (자동 못 잡는 것)

#### 1. 톤·연출 의도

자동 QC는 *기술적 무결성*만 본다.
*"이 컷이 채널 톤에 맞는가"*는 사람만 안다.

체크리스트:
- 컷의 *정서가 채널 정체성*에 맞는가 (자극·과장 톤 침범했는가)
- 마무리 사인오프(*"다시 본다"* 등)가 자연스럽게 들어왔는가
- BGM·SFX가 EBS다운가 (영화 BGM 톤으로 흘러가지 않았는가)

#### 2. 서사 호흡

12컷이 *15초 × 12 = 180초*로 산수상 맞아도, 보면 *호흡이 어색한 컷*이 있다.
긴 컷은 *왜 긴지*가 시청자에게 전달돼야 한다. 짧은 컷은 *짧은 만큼 정보 밀도*가 있어야 한다.

체크:
- 컷 1번부터 12번까지 *눈을 떼지 않고* 볼 수 있는가
- *지루한 컷*이 있으면 어디인가, 왜인가
- *너무 빠른 컷*이 있으면 어디인가, 정보 처리 가능한가

#### 3. 한국어 억양

ElevenLabs v3 한국어는 발음 OK / 억양·끊어읽기 약하다.
Seedance 2.0의 한국어 립싱크는 짧은 문장에서만 안정.

자동 QC는 *동기 오프셋*만 잡는다. *억양이 자연스러운가*는 사람만 안다.

체크:
- 어색한 강세·끊어읽기가 있는가
- 5-6 음절/초 톤이 컷마다 일정한가 (어떤 컷만 4 음절/초 느림)
- *"다시 본다"* 같은 사인오프가 *EBS다운 톤*으로 들리는가

#### 4. 문화 맥락

AI 생성 이미지는 *서양 미학으로 처리하는 경향*이 있다.
한국 정서가 짙은 장면(이사 짐 싸기, 한복, 1980년대 한국 풍경 등)에서 *서양 톤*이 침범했는가.

체크:
- 인물 표정·복장·배경이 *한국 시청자에게 자연스러운가*
- 1989년 KAIST 같은 *시대 고증*이 정확한가 (1989년에 LCD 모니터 X, CRT 모니터)
- 한국 풍경(거리·간판·자동차)이 *시기·지역에 맞는가*

#### 5. 채널 브랜드 톤

채널 사인오프, 톤, 색감, BGM 결이 *시즌 8편 전체*에 일관된가.
1편이 좋아도 2-8편과 톤이 분열되면 *채널이 안 만들어진다*.

체크:
- 이번 1편이 시즌 8편 전체의 *톤 표준*이 될 수 있는가
- 다음 7편을 같은 톤으로 만들 수 있는 *명시 가이드*가 정리됐는가 (`brand_assets/`)

### 재작업 결정 트리

자동 QC 통과 + 사람 검수에서 발견 시:

```
환각·사실 오류  → [제-3] 콘티 재수정 (가장 상류)
얼굴 불일치     → [제-4] 재생성 (LoRA 시드 고정)
립싱크 불량     → [제-5] 후처리 (Hedra 재합성)
컬러 일관성     → [제-5] 후처리 (Color Match)
톤·연출 부적절  → [제-3] 콘티 + [제-4] 재생성 ★ (가장 비싸다)
자막 동기 오류  → [제-5] WhisperX 재정렬
```

→ 가장 상류 단계로 한 번만 되돌리기. 무한 루프 금지.
→ *톤·연출 부적절*은 가장 비싸다. [제-3] 콘티 단계에서 *톤 명시*가 부족하면 여기서 무한 재작업이 생긴다. → 가설 1(컷-레벨 디렉터)의 핵심 작업이 [제-3]에 있는 이유.

## 인간 개입 4곳의 공통 원칙

### 1. *왜* 그 결정을 내렸는지 항상 적어라

```json
{
  "intervention_id": "[2]-cycle-001",
  "decision": "...",
  "rationale": "..."
}
```

→ 다음 사이클에서 페르소나 system prompt 개선의 input이 된다.

### 2. 페르소나의 영역을 침범하지 마라

[2]에서 *PD 대신 안건을 직접 만들지 마라*.
[5]에서 *CP 대신 점수를 직접 매기지 마라*.
[8]에서 *전사 대신 액션 시트를 직접 짜지 마라*.
[제-6]에서 *영상을 직접 편집하지 마라* (편집자 페르소나의 영역).

→ 사람이 페르소나 영역을 침범하면, 다음 사이클에서 페르소나가 *왜 약해졌는지* 분석할 수 없다.

### 3. 4곳 외에서는 사람이 끼지 마라

원칙적으로. 끼어들면 *시뮬레이션 결과의 신호*가 흐려진다.
다만 *페르소나 자체가 약한 신호*가 보이면 사이클 끝나고 system prompt 갱신 → 다음 사이클.

### 4. 인간 개입 4곳은 *PD의 일*이지 *사용자의 일*이 아니다

이 저장소를 받아쓰는 사람은 PD다. 사용자가 아니다.
4곳에서 결정하는 것은 *PD가 일하는 방식*의 본질이다. 다른 단계는 페르소나에게 위임할 수 있어도, 4곳은 못 위임한다.

→ 이게 *AI 시대 PD의 일*이 무엇인지에 대한 이 저장소의 자세 제안.
