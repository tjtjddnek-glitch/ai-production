- CP top5 추천 5개 중 2개만 일치 (PD2-5, PD5-1)

## 메타 가설 2의 자연 캐치 데이터

`04_human_cut.json`의 `insight_signals_for_report` 필드 직접 인용:

> 6. **메타 가설 2번 ('워크플로우 설계 = PD 채용') 자연 검증 데이터 첫 캐치 — CP·인간 평가 기준 차이가 명확히 드러남**
> 7. 다음 사이클 페르소나 system prompt 조정 시그널: CP에 '시리즈 일관성·테마 묶기'를 6번째 평가 축으로 추가 검토 가치

## 이 단계의 의미

이 단계가 메타 가설 2의 *데이터 자연 발생점*이다. 의도하지 않았던 룰 깸이 일어났고, 그 깸의 결이 *시리즈 일관성*이라는 6번째 평가 축으로 명료화됐다.

**다음 사이클 액션**:
1. CP system prompt에 6번째 축 "시리즈 일관성·테마 묶기" 추가 후 재실험 — overlap이 60%+가 되는가
2. PD A/B/C 페르소나 차이 실험 시 인간 평가에 6축 모두 반영해 평가 (현재는 5축)
3. 만약 6축 추가 후에도 overlap 40% 그대로면 → CP·인간이 *서로 다른 평가 함수*를 가져야 한다는 운영 결론 (둘 다 살린다)
