# 01. 리서치 단계 — 사회 시그널 + 도구 동향

> 30개 로그라인이 나오기 전에, 리서치 페르소나가 무엇을 모았는가.
> 이 단계의 산출물(`01_research.json`)이 이후 모든 PD 5명의 입력이 된다.

---

## 단계 위치

```
[1] 리서치 ★  ← 이 단계
[2] 가기획 로그라인 (PD 5명 × 6 = 30개)
[3] CP 평가
[4] 인간 픽
```

---

## 결정 narrative

리서치 페르소나는 두 종류의 입력을 동시에 모으는 역할이다:

1. **사회 시그널** — 한국 사회에서 지금 *흔들리는 것*. PD가 콘텐츠로 응답할 만한 가치 있는 질문이 어디서 발생하는가.
2. **도구 동향** — AI 영상 생성·천체역학·데이터 시각화 도구의 2026년 5월 시점 실제 가능치. 환각 위험·비용·한국어 적합성 포함.

이 두 입력을 한 JSON에 묶어 PD에게 넘기면, PD는 "도구가 가능한 일"과 "사회가 묻는 일"을 교차해서 로그라인을 짠다. 단순 트렌드 노트가 아니라 *작업 가능한 입력*이라는 게 핵심.

---

## 산출물

| 파일 | 내용 |
|---|---|
| `01_research.json` | 사회 시그널(인구·노동·기후·AI 격차) + 2026.5 도구 동향(Sora 2, Kling 2.5, Flux Pro 1.1, Hedra Character-3, ElevenLabs v3, Suno v5) + 1차 사료 가이드(국가기록원·국사편찬위·KOSIS·OECD Stats·NASA JPL Horizons 등) |

JSON 스키마 (요약):
```json
{
  "social_signals": [
    {"label": "...", "evidence": ["..."], "implication": "..."}
  ],
  "tooling_landscape": [
    {"tool": "...", "version": "...", "use_case": "...", "limits": "..."}
  ],
  "primary_sources": [
    {"name": "...", "url": "...", "use_case": "..."}
  ]
}
```

## 이 단계의 의미

이후 단계의 30개 로그라인 품질은 거의 전적으로 이 단계의 입력 폭에 의존한다. 같은 PD라도 *얕은 리서치 + 풍부한 리서치*에서 나오는 결과 차이가 메타 가설 2(워크플로우=채용)의 핵심 변수다.

다음 사이클에서 검토할 변수: 리서치 페르소나의 *사회 시그널 vs 도구 동향* 비중. 현재는 둘이 거의 같은 분량 — 하나를 더 두껍게 했을 때 PD 로그라인의 *novelty score*가 어떻게 달라지는가.
