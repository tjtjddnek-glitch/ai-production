## 컨셉 진화 — 무엇이 바뀌었는가

이 사이클의 본질은 **단발 7분 다큐가 시즌형 4부작으로 진화**한 것이다. 5번의 회의록이 그 진화의 흔적이다.

```
[2026-05-03~04]
  단발 7분 단편 다큐 (PD2-5 1438년의 별)
        ↓
[2026-05-05]  채널 부재 → "60초 6편 동시 업로드" 전략
  60초 단발 × 6편 (점심값·헌법·1438별·보이지않는산·AI노년·활판주조)
        ↓
[2026-05-06 Conference 002]  "그래서 어쩌라고?" 폐기 폭격
  살아남은 결: 60초 입구 + 시즌형 후속 가능 주제만
        ↓
[2026-05-06 Conference 003]  "더 깊게 가야겠다" → 3분 12컷
  3분 다큐 (잉크 세 번의 망설임 — 콘티 단계 픽션 발각, 폐기)
        ↓
[2026-05-07 Conference 004]  사실 검증 1순위 도입
  3분 다큐 + 1차 사료·논문 의무 (1821 측우기, "마음" 약함 폐기)
        ↓
[2026-05-07 Conference 005]  "마음 직격"으로 결 옮김
  3분 다큐 (33초 횡단보도 — 김진혁 결, 매일 보지만 묻지 않은 숫자)
        ↓
[2026-05-07 Conference 006]  사용자 직격 질문 ★ 최종 픽
  "한국 반도체 활황은 이대로 갈 것인가" → 시즌형 4부작 + 1부 12컷 콘티
  → "다시본다" 채널 + "한국의 시계" 4부작 (1부 우리 / 2부 중국 / 3부 AI / 4부 다음)
```

## 이 사이클에서 배운 것 3가지 (인사이트 후보)

### 1. 컷-레벨 디렉터 가설 — 콘티 픽션 사고가 가설 검증

**메타 가설 1**: "AI 제작 PD = 컷-레벨 디렉터. 책임 무게중심이 콘티 단계로 이동."

Conference 003 → 004의 폐기 사건이 직접 검증했다. 3분 12컷 풀 콘티가 *컴팩트한 픽션*이 될 수 있다 — 잉크 두께 11/19/27μm, 30분/2시간/다음날 같은 수치는 모델이 "그럴듯하게" 생성한 콘티 단계 환각. *현장이 없으니 콘티가 곧 사실*이고, 콘티가 픽션이면 영상 전체가 EBS 신뢰성 직격이 된다.

이후 Conference 004의 1원칙: **모든 수치는 논문 또는 1차 사료에서 가져온 것만**. 콘티에 출처 없는 수치 진입 금지.

→ "콘티가 사실의 닻"이라는 컷-레벨 디렉터 가설의 직접 증거.

### 2. 워크플로우 = PD 채용 가설 — 평가 기준 차이의 자연 캐치

**메타 가설 2**: "워크플로우 설계 = PD 채용 / 운영 = 정해진 워크플로우."

`04_human_cut.json`이 이 가설의 첫 자연 캐치 데이터다.

> "인간 픽 7개 중 4개가 CP에서 OK 또는 WEAK (PD1-5, PD3-1, PD3-6, PD4-1) — 인간은 정량 점수보다 직관·시리즈 의도 중시 / PD3 3개 통과(PD3-1·5·6 모두 인구·고령·세대 결)는 시리즈 묶기 의도가 룰을 압도함을 보여줌"

CP·인간 평가 overlap 40%. CP는 AI 필연성·STRONG 점수, 인간은 시리즈 일관성·테마 묶기. **다음 사이클 페르소나 system prompt 조정 시그널**: CP에 "시리즈 일관성·테마 묶기"를 6번째 평가 축으로 추가 검토 가치.

→ 다른 페르소나 PD = 다른 평가 가중치 = 다른 결과. PD를 "채용"한다는 의미가 자연 검증.

### 3. 진화는 강제 → 결의 누적이다

5번의 회의록은 매번 **금지 결**(폐기 사례)과 **원하는 결**(살아남은 룰)을 누적한다. 콘티 단계 픽션 한 번에 *사실 검증 1원칙*이 추가되고, "마음 직격 약함" 한 번에 *집단 정서·자기 직격*이 추가된다.

회의록 5개를 거치며 누적된 룰:
- (Conf 002) 데이터 환산이 *전복적*이어야 한다
- (Conf 002) 익숙한 사실 + 추상 메시지 X
- (Conf 003) 작은 사물 → 깊은 검증 → 반전 발견 결
- (Conf 004) 모든 수치는 1차 사료/논문 인용
- (Conf 005) 매일 보지만 왜 그런지 묻지 않은 숫자
- (Conf 005) *마음* 직격 (집단 정서 + 자기 좌표)
- (Conf 006) 단정 X — 조건부 시나리오 3개
- (Conf 006) 사용자 직격 질문이 시즌 구조 만듦

이 누적이 **다음 사이클의 평가 함수**가 된다. 페르소나 system prompt에 "이전 사이클 룰 N개"를 주입하면 다음 사이클의 폐기율은 줄고 픽 품질은 올라간다 — 이것이 *워크플로우 = 채용*의 운영 시그널.
